NombaRecur Documentation

Everything you need to integrate multi-rail recurring billing into your platform.

Go to Dashboard

Overview

NombaRecur acts as an abstraction layer between your platform and the Nomba API. Instead of building complex cron jobs, tracking failed charges, and handling Virtual Account Webhooks yourself, NombaRecur does it all for you.

Multi-Rail Fallback: If a customer's tokenized card charge fails (or if they use a Verve card which requires OTP), NombaRecur automatically falls back to a dedicated Virtual Account assigned to that customer.

1. Node.js SDK Installation

We provide a lightweight Node.js SDK to interact with your NombaRecur instance.

npm install nombarecur-js

2. Quickstart

Initialize the SDK using the Tenant API Key found in your NombaRecur Dashboard.

import { NombaRecur } from "nombarecur-js"; // Initialize with your API Key and your NombaRecur Base URL const client = new NombaRecur( "tenant_xxxxx_yyyyyy", "https://nomba-recur.vercel.app" ); async function createSub() { // 1. Create a customer const customer = await client.createCustomer({ email: "customer@example.com", name: "Jane Doe" }); // 2. Create a billing plan const plan = await client.createPlan({ name: "Pro Tier", amount_kobo: 500000 // 5,000 NGN }); // 3. Subscribe customer to the plan const sub = await client.createSubscription({ customer_id: customer.id, plan_id: plan.id, callback_url: "https://your-website.com/payment-success" }); console.log("Redirect user to:", sub.checkout_url); }

3. Canceling Subscriptions

You can easily cancel an active subscription at any time using the SDK.

await client.cancelSubscription("sub_xxxxxxx");

4. Webhooks

NombaRecur manages all Nomba webhooks internally to automatically reconcile billing state. You do not need to listen to Nomba webhooks directly.

Dashboard Integration: Just log into the NombaRecur Dashboard to see real-time MRR, active subscribers, and a live feed of all successful and failed billing attempts!
